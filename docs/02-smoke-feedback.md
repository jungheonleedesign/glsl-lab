# 2. 피드백 연기 — `FluidSim.tsx`

curl 속도장으로 연기를 흘려보내되, **이전 프레임을 기억**해 누적시킨다.

## 핵심 — ping-pong 버퍼 (기억)

셰이더는 같은 텍스처를 읽으며 동시에 쓸 수 없다. 그래서 버퍼 2장을 번갈아 쓴다.

```
프레임 N:   A(이전) ──읽기──> [시뮬] ──쓰기──> B,  B를 화면에
프레임 N+1: B(이전) ──읽기──> [시뮬] ──쓰기──> A,  A를 화면에
            └──────────── 역할 swap ───────────┘
```

이게 "기억"을 만든다. 이번 프레임이 **이전 프레임 위에 쌓이니** 연기가 누적·확산된다.
입자 시스템, 라이프 게임, 반응-확산, 물결 모두 이 구조 위에 올린다.

## advection — 흐름 따라 끌어오기 (semi-Lagrangian)

각 픽셀이 묻는다: **"지금 이 자리 연기는 한 프레임 전 어디 있었나?"**

```glsl
vec2 src = uv - vel * dt;            // 흐름 거슬러 한 프레임 전 위치
float density = texture2D(uPrev, src); // 거기 있던 값을 끌어옴
```

왜 '끌어오기'(뒤에서 가져옴)이고 '밀어내기'가 아닌가:
GPU 픽셀은 자기 값만 쓸 수 있고 남의 픽셀엔 못 쓴다 → 역방향 추적만 가능.
`src`가 픽셀 중앙이 아니어도 LinearFilter(쌍선형 보간)가 부드럽게 읽어준다.

## 속도장 = 노이즈 + 힘

```
vel = curl(p)          ① 소용돌이 난류 (스칼라 fbm → 90도 회전)
vel += buoyancy        ② 부력 (위로)
vel += mouseForce      ③ 마우스 힘
```

벡터라 그냥 더하면 합성된다(힘의 합성). 노이즈는 그중 '난류' 성분.

## 왜 Jacobi(압력)가 없나

curl 속도장은 **태생부터 비압축**이라 압력으로 강제할 필요가 없었다.
부력·마우스를 더하며 약간의 압축이 생기지만 — 연기는 그 오차가 **눈에 안 띄어**
그냥 용인했다. (물은 못 용인 → [4단계](04-fluid-solver.md)에서 압력 투영 등장.)

## 한계

매 프레임이 이전 위에 쌓이므로 진짜 누적은 되지만, 속도장이 절차적(curl)이라
중력·벽 같은 경계엔 정확히 반응하지 않는다. 진짜 유체는 압력 솔버가 필요하다.
