# 1. 노이즈 — `ShaderPlane.tsx`

"자연스러운 무작위"를 만드는 도구. 구름·물·불·지형이 전부 여기서 나온다.

## random — 가짜 난수

GPU엔 `rand()`가 없다. **해시 함수**로 "결정적이지만 무작위처럼 보이는" 값을 만든다.

```glsl
fract(sin(dot(p, vec2(12.9898, 78.233))) * 43758.5453)
```

원리: **엄청 민감한 함수에 넣고 정수부를 버려(fract) 소수부만 취한다.**
입력이 미세하게 변해도 소수부는 완전히 딴 값 → 무작위처럼 보임.
진짜 난수는 아니고 같은 입력엔 항상 같은 출력(hash).

## perlin noise — 격자 + 보간

화이트노이즈(모든 픽셀 난수)는 너무 거칠다. 대신 **격자점에만 정보를 두고 사이를 부드럽게 보간**한다.

- value noise: 격자점에 '값'을 둠
- perlin(gradient) noise: 격자점에 '기울기 방향'을 둠 → 격자 무늬가 덜 보임

```
floor(p) = 어느 칸인가 (정수)
fract(p) = 칸 안 위치 (0~1)   ← 좌표를 칸으로 쪼개는 기본 패턴
```

보간 비율은 **smoothstep `f*f*(3-2f)`** 으로 — 양 끝 기울기가 0이라
칸 경계에서 매끄럽게 이어진다 (직선 보간이면 각진 줄무늬가 생김).

## fbm — 옥타브를 겹쳐 디테일

노이즈 한 장은 밋밋하다. 주파수 2배·진폭 절반으로 여러 장(옥타브) 겹친다.

```
fbm = Σ amp·noise(freq·p)
  매 옥타브: freq ×2 (lacunarity), amp ×0.5 (gain)
```

큰 굴곡 위에 작은 굴곡 → 구름·산·물의 만능 베이스.
**더하기**라 같은 성격의 디테일이 양적으로 쌓인다.

## domain warping — 좌표를 노이즈로 비틀기

`fbm(p)` 대신 `fbm(p + fbm(p))`. 값이 아니라 **읽는 좌표(공간)를 휜다.**

```
워핑 0: 둥근 얼룩 (흐름 없음)
워핑 1: fbm(p + q)        → 흐름
워핑 2: fbm(p + r(q))     → 흐름 속 흐름 (곡률이 곱해짐)
```

**함수 합성**이라 디테일이 양적이 아니라 질적으로 달라진다(휘어짐).
fbm(더하기) vs warping(합성)의 차이가 핵심.

## curl — 스칼라 노이즈를 소용돌이 속도장으로

perlin/fbm은 **스칼라**(지형/높이). 속도(벡터)로 쓰려면 변환이 필요하다.

```
기울기 ∇ψ        : 등고선과 수직 (언덕 오르내림)
90도 회전 (∂ψ/∂y, -∂ψ/∂x) : 등고선을 따라 도는 흐름 = 소용돌이
```

90도 회전이 핵심. 안 돌린 기울기를 쓰면 솟고 빨려드는(발산) 가짜 흐름,
돌리면 **비압축(divergence-free) 소용돌이** → 진짜 유체처럼 보인다.
