# 5. 표면 위 유체 — `FluidMesh.tsx`

2D 유체 솔버를 구의 UV 공간에서 돌리고, 결과를 3D 표면에 입힌다.

## 핵심 — 시뮬 공간 ≠ 표시 공간

```
시뮬 (2D 텍스처)          표시 (3D 곡면)
  유체 솔버가 평면서 돎  ──UV 매핑──>  구 표면에 입혀짐
```

흐름은 **시뮬(텍스처 안 속도장)** 에서 일어나고, UV 매핑은 그 살아있는 텍스처를
곡면에 입히는 **영사기**일 뿐이다. (TV를 오브젝트에 감싸도 흐르는 건 영상이지 감싸기가 아님.)
입력은 raycast가 표면 클릭점의 uv를 주고, 그 uv에 splat한다.

## 두 가지 근본 난점

```
① UV 이웃 ≠ 표면 이웃 (seam)
   평면서 옆 칸이 3D 표면선 안 붙어있을 수 있음 → 이음새서 흐름 끊김.
   극점에선 UV가 한 점으로 수렴 → 잉크가 쥐어짜짐.

② '아래'가 점마다 다름
   평면 중력은 한 방향(-y)인데, 구 표면의 '아래'는 점마다 다름.
   진짜 흘러내림은 각 점의 접평면에 중력을 투영해야 함(접선 중력).
```

그래서 "표면서 휘젓고 번지기"는 바로 되지만, "중력 흘러내림"은 추가 작업이 필요.

## vertex displacement — 형태를 변형

지금까진 픽셀 색만 정했다. 이번엔 **vertex shader에서 텍스처를 읽어 정점을 민다.**

```glsl
float h = length(texture2D(uDye, uv).rgb);   // 물 두께 = 색 밝기
vec3 displaced = position + normal * h * uDisplace;
gl_Position = ... * vec4(displaced, 1.0);
```

보통 텍스처는 fragment(색)에서 읽는데, 여기선 **vertex에서 읽어 위치(기하)** 를 정한다.
물이 고인 곳이 볼록해지고 silhouette(외곽선)가 변한다 — '기하가 반응'.

## 한계 — 표면에 묶임

법선 방향으로 밀 뿐이라 물이 표면을 **떠날 수 없다**.
```
가능: 표면이 볼록, 물줄기 도드라짐
불가: 떨어져 나가 줄줄/뚝뚝
```
떨어지는 물은 표면에 묶이지 않은 독립 표현이 필요 → [6단계](06-water-particles.md).

→ 표현 방식 자체가 다르다 (텍스처가 아니라 3D 입자).
