# 6. 입자 물 — `WaterParticles.tsx`

물을 표면 텍스처가 아니라 **3D 공간을 떠다니는 입자 수만 개**로 표현한다.
그래야 표면을 떠나 떨어질 수 있다. 배운 것 둘의 합체.

## 표현 방식 — 표면 묶임 vs 자유 부피

```
1단계 색 흐름 / 2단계 변위 : 물이 구 표면(UV)에 묶임 → 못 떨어짐
3단계 입자                : 물이 자기 3D 좌표를 가짐  → 떨어지고 튐
```

떨어지는 물방울을 원하면 무조건 자유 부피(입자) 표현.

## GPGPU 입자 — ping-pong을 필드 대신 입자에

입자 상태(위치·속도)를 텍스처에 저장. **텍셀 1개 = 입자 1개.**
셰이더가 모든 입자를 병렬 갱신, ping-pong으로 이전→다음.

```
velPass: vel += gravity·dt;  충돌 반사
posPass: pos += vel·dt;      하드 구속 + 수명/재생성
```

충돌(구): 안으로 파고드는 속도 성분만 제거 → 표면 따라 흐르다 가장자리서 떨어짐.

```glsl
vec3 n = normalize(pos); float vn = dot(vel, n);
if (vn < 0.0) vel -= (1.0+restitution)*vn*n;   // 내적으로 법선/접선 분해
vel *= friction;
```

## vertex 텍스처 읽기 — 입자를 그리기

점 N개를 그리되 각 점이 **위치 텍스처에서 자기 좌표를 읽는다** (5단계 변위와 같은 기법).

```glsl
vec4 p = texture2D(uPos, ref);   // ref = 이 점의 텍셀 좌표
gl_Position = projectionMatrix * modelViewMatrix * vec4(p.xyz, 1.0);
```

= 연기의 **ping-pong** + 구 변위의 **vertex 텍스처 읽기**. 둘이 여기서 만난다.

## 스크린스페이스 유체 렌더 — 입자를 물처럼

입자 점으로 그리면 알갱이로 보인다. 매끈한 물로 보이려면:

```
③ 배경 솔리드(구·바닥) → sceneTarget
④ 입자를 부드러운 블롭으로 가산 누적 → '두께'
④-b 두께를 블러 (가우시안) → 봉우리를 뭉개 매끈한 표면  ← 결정적 단계
⑤ 합성:
   법선  = 두께 기울기 (5단계 변위 기법)
   색    = exp(-두께) Beer-Lambert (3단계, 두꺼울수록 짙은 파랑)
   굴절  = 배경을 법선만큼 휘어 비춤
   프레넬 + 스페큘러
```

블러가 핵심. 안 하면 입자마다 봉우리가 남아 알갱이로 보인다.
`useFrame(..., 1)`의 **priority 1** 이 R3F 자동 렌더를 끄고 이 멀티패스를 직접 제어한다.
