# 4. 유체 솔버 — `FluidSolver2D.tsx`

진짜 비압축 유체. 연기에서 피했던 **압력 투영**을 직접 구현한다.
(C4D 같은 툴이 안에서 해주던 그 계산.)

## 전체 루프 (Stam, Stable Fluids)

```
1. 힘 추가      중력·마우스를 속도에 더함
2. 이류         속도가 자기 자신을 따라 흐름 (semi-Lagrangian, 2단계서 배움)
3. 압력 투영 ★  속도를 비압축으로 강제 (새 심장)
4. 이류         물색(dye)을 속도 따라 옮김
```

1·2·4는 이미 아는 것. **3번만 새것**이고 이게 유체의 전부다.

## 압력 투영 — 새 심장

이류·중력 뒤 속도장은 엉망(어떤 곳은 모이고=압축, 어떤 곳은 비고=팽창).
실제 물은 압축 안 되니 이 **발산을 0으로** 강제해야 한다.

```
발산 div = 이 칸에 얼마나 모여드나/빠져나가나
  div=0 이면 비압축 (들어온 만큼 나감)
```

헬름홀츠 분해: 아무 속도장 = (비압축 부분) + (∇p). 군더더기 ∇p만 빼면 깨끗.
p는 포아송 방정식으로 구한다:

```
∇²p = div(속도)
  "어디가 얼마나 압축됐나"를 알면 "그걸 풀 압력 p"를 역으로 구함
```

## Jacobi 반복 — 압력 풀기

`∇²p = div`를 한 번에 못 푼다. **이웃 평균을 반복**해 압력이 격자 위로 번지게 한다.

```
새 압력 = (상하좌우 압력 합 - 발산) / 4    ← 한 번
20~40회 반복 → 압력이 퍼져 평형
```

그다음 `v = v - ∇p` 로 군더더기를 빼면 비압축 완성.

## construct vs project (연기와의 차이)

```
연기: curl로 비압축을 '처음부터 구성'   → 압력 솔버 불필요
물:   임의 힘(중력·벽)으로 엉망이 된 흐름을 '비압축으로 투영' → Jacobi 필수
```

물은 임의 힘 때문에 구성으로 못 만들고, 매 프레임 엉망을 되돌려야(투영) 한다.

## 멀티패스 (GPU)

```
버퍼: 속도(2) · 압력(2) · 발산(1) · 물색(2)   전부 ping-pong
패스: ①이류 → ②힘splat → ③발산 → ④Jacobi×30 → ⑤기울기빼기(+벽) → ⑥색이류 → ⑦표시
```

⑤에서 가장자리 속도를 0으로 만들어 유체를 벽 안에 가둔다.

## 확인 실험
